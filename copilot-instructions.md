# Copilot Agent System — Copilot Instructions

> This repository (`.github-private`) contains the shared AI agent system for the Copilot Agent System platform.
> It is consumed org-wide by all repositories in the Org GitHub Enterprise organization.

## How This Repo Works

- **Agents** (`agents/`) — Shared org-wide automatically. All repos in the org see these agents.
- **Prompts** (`prompts/`) — Shared org-wide. Users can also clone this repo and symlink the folder for local access.
- **Skills** (`skills/`) — Shared org-wide. Users can also clone and symlink for local access.
- **Instructions** (`.github/instructions/`) — Auto-injected based on `applyTo` glob patterns when editing matching files.
- **Guardrails** (`.github/instructions/GUARDRAILS-*.instructions.md`) — Three-tier safety system loaded automatically.

## Key Rules

- All agents follow the guardrails defined in `.github/instructions/GUARDRAILS-core.instructions.md`
- Code-specific rules are in `GUARDRAILS-code.instructions.md` (loaded for `.cs`, `.ts`, `.tsx` files)
- Orchestration and chain rules are in `GUARDRAILS-orchestration.instructions.md` (loaded for `agents/**`)
- MCP servers (Jira, Confluence) are configured at the organization level — see the Developer Manual for setup details

## Dependency Change Policy

Any new npm or NuGet dependency added by an agent requires explicit user confirmation with justification before installation. Agents must not silently add packages.

## Memory & Context Retention

This repo uses a two-layer memory system to reduce context loss across sessions:

- **Org-level** (`memories/org/`): Shared conventions, anti-patterns, and ADRs — check before architectural decisions.
- **Repo-level** (`.copilot/memories/repo/` in each project): Session summaries and gotchas auto-generated by the session-logger hook.

## Knowledge Base

Domain knowledge for each application is maintained as a living knowledge base generated by `/knowledge-init`:

- **`{folder}/AGENTS.md`** — 1-page domain index per feature/domain folder. VS Code Copilot reads this automatically. Contains: purpose, key entities, entry points, key files, links to deep docs.
- **`docs/{domain}.md`** — Deep knowledge doc per domain: bounded context, data model, business rules, flows, external contracts, gotchas.
- **`docs/INDEX.md`** — Master index with project metadata (Value Stream, Confluence URL, Jira keys/epics).

The `knowledge-drift` hook runs at `sessionEnd`, detects source file changes, and flags stale domain docs in `.copilot/memories/repo/knowledge-drift.md`. Run `/knowledge-init --update` to refresh stale domains.

> **Knowledge only** — AGENTS.md files contain domain facts, not coding standards. Standards live in GUARDRAILS.

### Agent Responsibilities

1. At session start: check for recent session summaries in `.copilot/memories/repo/session-summaries/` for continuity.
2. At session start: check `.copilot/memories/repo/knowledge-drift.md` for stale domain docs — surface to user if any exist.
3. During work: use keywords (`gotcha:`, `lesson:`, `remember:`, `important:`) so the hook captures learnings.
4. Before architecture decisions: consult `memories/org/conventions.md` and `memories/org/architecture-decisions.md`.
5. When patterns recur: suggest promoting repo-level gotchas to org-level via PR.

See `.github/instructions/memory-conventions.instructions.md` for full rules.
