# Org-Wide Architecture Decisions

> ADRs that apply across all Copilot Agent System projects.

## ADR-001: Shared Agent System via awesome-skills-copilot

- **Context**: Multiple teams need consistent AI assistance across 20+ repos.
- **Decision**: Single `awesome-skills-copilot` repo with agents, skills, instructions, and hooks shared org-wide.
- **Consequence**: All changes to AI behavior are centralized; individual repos can override thresholds only.

## ADR-002: Two-Layer Memory Architecture

- **Context**: Developers lose context between sessions; learnings in one project don't reach others.
- **Decision**: Org-level memories in `awesome-skills-copilot/memories/org/` + repo-level in `.copilot/memories/repo/`.
- **Consequence**: Org memories are shared via git (this repo). Repo memories are local and auto-generated by hooks.

## ADR-003: Hook-Based Session Persistence

- **Context**: VS Code Copilot sessions are ephemeral; no built-in cross-session memory.
- **Decision**: Use session-logger hook to generate session summaries at session end, stored in `.copilot/memories/repo/session-summaries/`.
- **Consequence**: Lightweight summaries persist in git. Full context still relies on Copilot's native `/memories/session/` during active sessions.
